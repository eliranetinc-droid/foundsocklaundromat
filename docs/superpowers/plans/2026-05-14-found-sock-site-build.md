# Found Sock Laundromat — Site Build Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a complete, SEO-perfect (100/100 Lighthouse mobile) static website for The Found Sock Laundromat, deployable to Cloudflare Pages at a temporary URL (`foundsock-new.pages.dev`), ready for the user to review on their phone before cutover from Wix.

**Architecture:** Astro 5 static-site generator with Tailwind CSS v4. Zero JavaScript shipped to the browser by default. All content in markdown + JSON. Single source of truth (`business.json`) drives NAP/hours/schema across the entire site. Three external integrations (Freshdesk REST API, Google Places API, Google Maps embed) all run at build time or via tiny per-page snippets.

**Tech Stack:** Astro 5, Tailwind CSS v4, TypeScript, Vitest (unit tests), Playwright (smoke tests), Lighthouse CI (perf regression), self-hosted Inter + Instrument Serif fonts, Cloudflare Pages.

**Spec reference:** [`docs/superpowers/specs/2026-05-14-found-sock-laundromat-website-design.md`](../specs/2026-05-14-found-sock-laundromat-website-design.md)

**Decomposition principles applied:**
- **Single source of truth** in `src/data/business.json` — every page reads from it
- **Components by responsibility, not layer** — `Hero.astro` owns the hero on the homepage; if it ever appeared elsewhere we'd extract; if a homepage block grows logic we'd give it its own file
- **`src/lib/*` for pure logic** — open-hours and schema generators are fully unit-tested; everything else is template
- **`/api/submit-ticket.ts`** is the only server-side code — protects the Freshdesk API token from being exposed in the browser

---

## Test strategy

**Test what has logic. Don't test what's just markup.**

- **Unit-tested (TDD, Vitest):**
  - `src/lib/open-hours.ts` — date/time math has real edge cases (timezone, midnight, day transitions)
  - `src/lib/schema.ts` — JSON-LD generators must produce schema-valid output
  - `src/pages/api/submit-ticket.ts` — input validation, error handling
  - `scripts/fetch-google-reviews.mjs` — handles API errors, malformed responses
- **Smoke-tested (Playwright):** Every page loads, key elements visible, no console errors
- **Performance-tested (Lighthouse CI):** Homepage + 3 representative inner pages must score ≥95 on mobile across all 4 categories (perf/a11y/best-practices/SEO)
- **Visual review:** User clicks through the temporary URL on their phone

**Pages themselves are not unit-tested.** They're declarative templates. Their test is "does the page build + does Lighthouse pass + does it look right."

```typescript
import { getBusiness } from './business';

const TZ = 'America/New_York';

export type OpenStatus = {
  isOpen: boolean;
  label: 'Open now' | 'Closed';
  detail: string; // e.g. "closes 11 PM" or "opens 6 AM"
};

function partsInTz(date: Date): { hour: number; minute: number } {
  const fmt = new Intl.DateTimeFormat('en-US', {
    timeZone: TZ,
    hour12: false,
    hour: '2-digit',
    minute: '2-digit',
  });
  const parts = fmt.formatToParts(date);
  const hour = Number(parts.find(p => p.type === 'hour')?.value ?? 0);
  const minute = Number(parts.find(p => p.type === 'minute')?.value ?? 0);
  return { hour: hour === 24 ? 0 : hour, minute };
}

function formatHour12(hhmm: string): string {
  const [hStr, mStr] = hhmm.split(':');
  const h = Number(hStr);
  const m = Number(mStr);
  const period = h >= 12 ? 'PM' : 'AM';
  const h12 = h === 0 ? 12 : h > 12 ? h - 12 : h;
  return m === 0 ? `${h12} ${period}` : `${h12}:${String(m).padStart(2, '0')} ${period}`;
}

export function getOpenStatus(now: Date = new Date()): OpenStatus {
  const { hours } = getBusiness();
  const { hour, minute } = partsInTz(now);
  const nowMinutes = hour * 60 + minute;

  const [oh, om] = hours.open.split(':').map(Number);
  const [ch, cm] = hours.close.split(':').map(Number);
  const openMinutes = oh * 60 + om;
  const closeMinutes = ch * 60 + cm;

  const isOpen = nowMinutes >= openMinutes && nowMinutes < closeMinutes;

  if (isOpen) {
    return { isOpen: true, label: 'Open now', detail: `closes ${formatHour12(hours.close)}` };
  }
  return { isOpen: false, label: 'Closed', detail: `opens ${formatHour12(hours.open)}` };
}
```

```typescript
export type TicketInput = {
  subdomain: string;
  apiKey: string;
  name: string;
  email: string;
  subject: string;
  description: string;
  type: 'issue' | 'general';
};

export async function submitFreshdeskTicket(input: TicketInput): Promise<{ id: number }> {
  const auth = btoa(`${input.apiKey}:X`);
  const body = {
    name: input.name,
    email: input.email,
    subject: input.subject,
    description: input.description.replace(/\n/g, '<br>'),
    priority: input.type === 'issue' ? 2 : 1,
    status: 2, // open
    type: input.type === 'issue' ? 'Issue Report' : 'General Inquiry',
  };

  const res = await fetch(`https://${input.subdomain}.freshdesk.com/api/v2/tickets`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Basic ${auth}`,
    },
    body: JSON.stringify(body),
  });

  if (!res.ok) {
    const text = await res.text();
    throw new Error(`Freshdesk error ${res.status}: ${text}`);
  }
  const data = await res.json() as { id: number };
  return { id: data.id };
}
```

```typescript
import type { APIRoute } from 'astro';
import { submitFreshdeskTicket } from '../../lib/freshdesk';

const isEmail = (s: string) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(s);

export const POST: APIRoute = async ({ request, locals }) => {
  let payload: Record<string, unknown>;
  try {
    payload = await request.json();
  } catch {
    return new Response(JSON.stringify({ error: 'invalid_json' }), { status: 400 });
  }

  const { name, email, message, type } = payload as Record<string, string>;
  if (!name || name.length < 1 || name.length > 100)        return errorJson('invalid_name');
  if (!email || !isEmail(email))                            return errorJson('invalid_email');
  if (!message || message.length < 1 || message.length > 5000) return errorJson('invalid_message');
  if (type !== 'issue' && type !== 'general')               return errorJson('invalid_type');

  const env = (locals as any).runtime?.env ?? import.meta.env;
  const subdomain = env.FRESHDESK_SUBDOMAIN as string | undefined;
  const apiKey = env.FRESHDESK_API_KEY as string | undefined;
  if (!subdomain || !apiKey) {
    return new Response(JSON.stringify({ error: 'server_misconfigured' }), { status: 500 });
  }

  try {
    const result = await submitFreshdeskTicket({
      subdomain, apiKey, name, email,
      subject: type === 'issue' ? `Issue from ${name}` : `Inquiry from ${name}`,
      description: message,
      type,
    });
    return new Response(JSON.stringify({ ok: true, ticketId: result.id }), { status: 200 });
  } catch (e) {
    console.error('Freshdesk submission failed', e);
    return new Response(JSON.stringify({ error: 'upstream_failed' }), { status: 502 });
  }
};

function errorJson(error: string): Response {
  return new Response(JSON.stringify({ error }), { status: 400 });
}
```

- [ ] **Step 2: Write fetch-google-reviews.mjs**

`scripts/fetch-google-reviews.mjs`:
```javascript
import fs from 'node:fs/promises';
import path from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const OUTPUT_PATH = path.join(__dirname, '..', 'src', 'data', 'reviews.json');

export async function fetchReviews({ apiKey, placeId }) {
  const url = `https://maps.googleapis.com/maps/api/place/details/json?place_id=${placeId}&fields=rating,user_ratings_total,reviews&key=${apiKey}`;
  const res = await fetch(url);
  if (!res.ok) return null;
  const data = await res.json();
  const r = data.result;
  if (!r) return null;
  return {
    rating: r.rating,
    count: r.user_ratings_total,
    fetchedAt: new Date().toISOString(),
    reviews: (r.reviews ?? []).slice(0, 5).map(rv => ({
      author: rv.author_name,
      rating: rv.rating,
      text: rv.text,
      relativeTime: rv.relative_time_description,
    })),
  };
}

async function main() {
  const apiKey = process.env.GOOGLE_PLACES_API_KEY;
  const businessJson = JSON.parse(await fs.readFile(path.join(__dirname, '..', 'src', 'data', 'business.json'), 'utf-8'));
  const placeId = businessJson.googlePlaceId;

  if (!apiKey || !placeId) {
    console.log('[fetch-google-reviews] Skipping — missing GOOGLE_PLACES_API_KEY or googlePlaceId in business.json. Existing reviews.json will be used.');
    return;
  }

  const data = await fetchReviews({ apiKey, placeId });
  if (!data) {
    console.warn('[fetch-google-reviews] API call failed. Keeping existing reviews.json.');
    return;
  }
  await fs.writeFile(OUTPUT_PATH, JSON.stringify(data, null, 2));
  console.log(`[fetch-google-reviews] Wrote ${data.reviews.length} reviews (rating ${data.rating}, ${data.count} total).`);
}

if (import.meta.url === `file://${process.argv[1]}`) {
  main().catch(err => { console.error(err); process.exit(1); });
}
```

## Task 28: Weekly review-refresh cron (Cloudflare Worker)

**Files:**
- Create: `.github/workflows/weekly-rebuild.yml`

Cloudflare Pages doesn't have built-in cron, but GitHub Actions does. The action calls Cloudflare Pages' deploy hook on a schedule, triggering a rebuild that re-fetches Google reviews.

- [ ] **Step 1: Create Cloudflare Pages deploy hook**

Cloudflare Dashboard → Pages → foundsocklaundromat → Settings → Builds & deployments → Deploy hooks → Add deploy hook. Name: `weekly-rebuild`. Branch: `main`. Save the URL it returns.

- [ ] **Step 2: Add the deploy hook URL as a GitHub secret**

GitHub → repo → Settings → Secrets and variables → Actions → New repository secret. Name: `CLOUDFLARE_DEPLOY_HOOK`. Value: the URL from Step 1.

- [ ] **Step 3: Create workflow file**

`.github/workflows/weekly-rebuild.yml`:
```yaml
name: Weekly Rebuild (refresh Google reviews)

on:
  schedule:
    - cron: '0 8 * * 1'  # Mondays 8am UTC = 4am ET (off-peak)
  workflow_dispatch:       # also run on-demand from Actions tab

jobs:
  trigger-rebuild:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger Cloudflare Pages deploy
        run: |
          curl -X POST -f "${{ secrets.CLOUDFLARE_DEPLOY_HOOK }}"
```

- [ ] **Step 4: Commit and test**

```bash
git add .github/ && git commit -m "ci: weekly rebuild to refresh Google reviews"
git push
```

After pushing, go to GitHub → Actions tab → Weekly Rebuild → Run workflow → verify a fresh deployment starts in Cloudflare Pages.
